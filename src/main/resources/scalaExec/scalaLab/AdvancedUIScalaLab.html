The ScalaLab Editor is a simple programmer's editor, that is integrated with the system. It provides support in the following basic ways:



a. Coloring of the text.

The Scala reserved words are automatically detected and are colored. Also, the basic library functions (e.g. fft(), linspace(), logspace(), inc() etc) are detected and are colored differently to aid the appearance of the text. Also, variables that currently have values defined are colored differently, in order to facilitate the programmer visually.

The coloring is accomplished with the aid of the JTextPane Java's Swing component that offers the functionality to implement effective cross-platform editors with advanced styled capabilities.

b. Autocompletion

The Scala interpreter offers an API that allows IDEs (Integrated Development Environments) to obtain information in order to implement code completion functionality. The editor supports TAB completion for global workspace variables and for the members of object variables.



c. Selective execution of program parts

The editor allows the selective execution of the code by providing options like (see also the popup menu):

* Execute the whole text in editor (F8)

* Execute the text from the beginning to the cursor location (F9)

* Execute the text up to the previous location (F7)

The executed parts of the program are marked gray. The user can reset the text coloring at any time by pressing F3.

This is accomplished by feeding the program text to the Scala interpreter for execution. We should note at this point that the integration with the Scala Compiler is not tight in the sense that the editor does not retrieve the ASTs (Abstract Syntax Trees) of the compiled code chunks. Such a design will be very complicated since it is not an easy task to map the compiled ASTs to source code lines.

d. Integration with both the Java and the Scala Compiler

The editor provides a popup menu option to compile and run the source text, both with the Scala compiler and the Java compiler, which one is appropriate. Therefore, the user is facilitated to develop both Java and Scala applications, that can utilize the ScalaLab libraries without the separate installation of external Java and Scala compilers.

e. Wizards for application types.

The editor provides skeleton for various application types that utilize standard toolboxes, e.g. BioJava application, WEKA application. The skeleton code is in the form of the construction of the proper import statements.





The AutoCompletion System

The autocompletion system keeps in a scanMethodsScalaSci Vector the items available for completion. That vector (i.e. scanNethodsScalaSci) is kept sorted in order to retrieve effectively the matched items.

The ListOfAllMethodsScalaSci array of Strings is set to the entries of the scanMethodsScalaSci array, therefore allowing to view them as a list.
