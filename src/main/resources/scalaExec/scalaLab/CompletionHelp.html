Tutorial on JSyntaxPane Completion


The JSyntaxPane editor supports two modes of code completion, the Global mode and the Scala mode.  ScalaLab exit and is available for the next working session.

The “Global”  completion mode is used with CTRL-SPACE.  It is useful to remind the large number of  routines of ScalaLab. Also,  many of these routines are overloaded and thus it is more difficult for the user to remember their usage. One good example is the plot() routines that has a lot of overloaded versions. The user can simply type plot and then CTRL-SPACE. The code completion feature of the JsyntaxArea editor reminds the user of all the overloaded versions of plot.

The “Scala” completion mode is used with F7. It can also be very useful for other completion tasks.  That mode extracts information using a Scala completer object. One very usefl task is package completion.  Suppose that we want to display the contents of the package org.ejml. We simple write: org.ejml.  and then we press F7.
The completer displays to us the contents of the package org.ejml  with an elegant popup completion list. 
Also, very convenient is the member completion feature. Suppose that we execute the command:
var xx = new JFrame("test")
Then we can easily remember the many fields/methods of the javax.swing.JFrame class, using:
xx. <F7>

Tutorial on RsyntaxArea Completion


The RSyntaxArea editor supports two modes of code completion, the Global mode and the Scala mode.  The menu “Completion” can be used to switch between these two completion modes, with the submenu option: 
“ Toggle Global/Scala completion mode, current mode is: .... “
That submenu also displays the current completion mode. Also the current completion mode is saved upon ScalaLab exit and is available for the next working session.

The “Global” completion mode is useful to remind the large number of  routines of ScalaLab. Also,  many of these routines are overloaded and thus it is more difficult for the user to remember their usage. One good example is the plot() routines that has a lot of overloaded versions. The user can simply type plot and then CTRL-SPACE, while RsyntaxArea is in “Global” completion mode. The code completion feature of the RsyntaxArea editor reminds the user of all the overloaded versions of plot.
Also, we can from the “Completion” menu expand the completion lists available at the “Global” completion mode using information extracted from libraries using Java reflection. 

The “Scala” completion mode can also be very useful for other completion tasks.  That mode extracts information using a Scala completer object. One very usefl task is package completion.  Suppose that we want to display the contents of the package org.ejml. We simple write: org.ejml.  and then we press CTRL-SPACE wile in “Scala” completion mode. The completer displays to us the contents of the package org.ejml  with an elegant popup completion list. 
Also, very convenient is the member completion feature. Suppose that we execute the command:
var xx = new JFrame("test")
Then we can easily remember the many fields/methods of the javax.swing.JFrame class, using:

F4 Completion
The F4 key works similar for both editors (i.e. JSyntaxPane and RsyntaxArea). It displays a completion list  with about the same contents as that obtained from Scala Completion but by using Java reflection.
