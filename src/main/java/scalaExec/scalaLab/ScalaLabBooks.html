ScalaLab Scientific Resources



ScalaLab includes a Java 6 compiler and thus it can compile and run Java code even without an installed Java compiler. Important and powerful Java scientific libraries have been included at the main ScalaLab distribution and others can be utilized with the toolbox loading mechanism.

There are two ways to run Java scientific code:



a. The traditional way as a Java class

Open the ScalaLab Editor (with “Ctrl-E” or “Edit”, “scalaLab Editor”) and implement a Java class. Then run the class with the “Java Compile and Execute (F9)” option

For example suppose that we want to use the Efficient Java Matrix Library (EJML) from within ScalaLab. Specifically, we run the BenchmarkElement class shipped with the EJML distribution. We proceed as follows.

Step 1. Open the ScalaLab Editor and code the BenchmarkElement class as follows.

import org.ejml.*;

import org.ejml.data.DenseMatrix64F;

import org.ejml.ops.CommonOps;

import org.ejml.ops.RandomMatrices;

import java.util.Random;

public class BenchmarkElement {

static Random rand = new Random(234);

public static void main( String args[] ) {

long N = 10000000;

double num = 2.5;

DenseMatrix64F A = RandomMatrices.createRandom(10,10,rand);

long timeBefore = System.currentTimeMillis();

for( int i = 0; i < N; i++ ) {

CommonOps.divide(num,A);

}

long timeAfter = System.currentTimeMillis();

System.out.println("div = "+(timeAfter-timeBefore));

A = RandomMatrices.createRandom(10,10,rand);

timeBefore = System.currentTimeMillis();

for( int i = 0; i < N; i++ ) {

CommonOps.scale(num,A);

}

timeAfter = System.currentTimeMillis();

System.out.println("scale = "+(timeAfter-timeBefore));

}

}

Step 2 Save the code in a file named BenchmarkElement.java

Step 3 Compile and Execute the Java code with the relative popup menu option or with the shortcut F9.



b. Scripting with the Scala Interpreter

This is the preferable way. Any Java code can be executed within the framework of the Scala Interpreter. The basic Java scientific libraries are within the ScalaLabLibs.jar file which exists in the lib folder after the installation of ScalaLab. Thus there are accessible from the ClassPath and can be directly used.

The following important Java libraries can be used:

a. The NUMAL numerical analysis Java library.

This library is described with the book:

“A Numerical Library in Java for Scientists and Engineers”, of Hang T. Lau, Chapman & Hall, 2004



b. The Java implementations of numerical code of the book:

“Object-Oriented Implementation of Numerical Methods”, of Didier H. Besset, Morgan Kaufman Publishers, 2001
